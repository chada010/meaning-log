# Resend SMTP 配置未加载

## 现象

前后端、MySQL 和 Redis 均正常运行，但注册或重置密码时无法发送邮箱验证码。相同 Resend 凭据曾通过 VS Code 启动配置成功发送。

## 真实数据

- 本机到 `smtp.resend.com:2465` 的 TCP 与 TLS 连接成功。
- SMTP `AUTH LOGIN` 返回 `235`，证明现有仅发信 Key 可用。
- 运行中的后端进程只收到 MySQL 与 Redis 环境变量。
- Resend 配置仅存在于被 Git 忽略的 `.vscode/launch.json`，统一启动脚本没有加载这些值。

## 根因

应用代码通过通用 `MAIL_*` 环境变量读取 SMTP 配置，但本地统一启动脚本没有传递邮件变量，导致后端退回 QQ 示例配置。直接把 Resend Key 加入原有 Base64 命令参数还会让秘密值出现在可检查的进程命令行中。

## 修复

- 根目录 `.env` 作为本地邮件配置来源。
- 启动脚本在启动服务前校验必填邮件变量，并拒绝示例发件域名。
- 后端在所有启动方式下对缺失或示例邮件配置 fail-fast。
- 后端子进程继承临时进程环境，不在命令行中携带秘密值。
- 子进程使用结构化工作目录和参数启动，不把文件路径拼入脚本文本。
- 启动成功后持久化应用进程身份，并提供统一停止入口。
- 旧版 Compose 数据卷可通过显式项目名覆盖继续复用。
- Spring Mail 使用 Resend 默认主机与端口，并配置有限的 SMTP 超时。

## 教训

第三方服务配置必须从标准启动入口做端到端验证。IDE 私有启动配置成功，不能证明项目统一启动脚本加载了相同配置；Base64 也不是秘密保护机制。
